# Pull Request Policy

This repository only accepts pull requests for links to applications that have real engineering and practical significance.

Projects created:

* "on the fly" in a couple of days,
* in vibecoding format without architecture, requirements, or justification,
* as "the coolest SCADA" without undergoing any design or operational testing,

will not be considered or accepted.

## What we consider a significant contribution

A pull request can be accepted if it:

* solves a real industrial or engineering problem
* has a clear description of the purpose and scope
* complies with the project's architectural principles
* is accompanied by documentation, tests, and rationale for decisions
* is designed for long-term support

## What is not accepted

* Experimental or demonstration "prototypes" without practical value
* "Quickly cobbled together" control, monitoring, or SCADA-like solutions
* Changes added solely for the sake of hype, trends, or personal portfolio

## Important

Engineering is a responsibility, not a vibecoding sprint.
